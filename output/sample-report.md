## 3. Hook 分析

### Pattern 1：反差式成果

- 代表貼文：@example 的 AI Agent 工作流貼文。
- 為什麼有效：先呈現「以前需要人工處理」與「現在可由 Agent 協作」的反差，讓讀者立刻理解收益。
- 可模仿句型：我把原本要花 `X 小時` 的工作，改成 `Y 步驟` 的 AI workflow，結果最意外的是 `Z`。

### Pattern 2：第一人稱踩坑

- 代表貼文：@creator 的 ChatGPT 工作流心得。
- 為什麼有效：Threads 上口語、真實、帶一點情緒的敘事，比硬教學更容易引發留言。
- 可模仿句型：我本來以為 `工具/方法` 只適合 `A`，直到我拿它處理 `B`，才發現真正省時間的是 `C`。

### Pattern 3：工具清單承諾

- 代表貼文：MCP 工具整理類內容。
- 為什麼有效：清單型內容收藏價值高，對開發者與創作者都容易轉發。
- 可模仿句型：如果你只想先試 5 個 AI workflow 工具，我會從這幾個開始。

## 4. 趨勢分析

- 今天明顯升溫的討論：AI Agent 不再只被包裝成概念，更多貼文開始展示具體工作流、工具組合與實作畫面。
- 可能延燒 3-7 天的話題：MCP server 推薦、AI coding agent 使用邊界、企業內部導入 AI workflow 的案例。
- 需要避開的過度擁擠角度：單純喊「AI 會取代人」或「這工具太神」已經疲乏，除非有具體數據、截圖或反直覺觀點。

## 5. 可模仿內容

### X 貼文 idea

1. 我把每天重複做的 3 個任務改成 AI Agent workflow，真正省時間的不是自動化，而是少掉這個環節。
2. 這 5 個 MCP server 適合剛開始做 AI workflow 的人，先不要一次裝太多。
3. 用 AI 寫程式最容易踩的坑：不是它寫錯，而是你沒有定義完成條件。
4. 我測了 3 種 AI coding 工作流，最穩的是「先規格、再實作、最後人工 review」。
5. 如果你今天才開始研究 AI Agent，先看這個最小可行流程。

### Threads 貼文 idea

1. 我最近發現，AI 工具真正改變工作的地方，不是幫你省下全部時間，而是讓你少卡在第一步。
2. 以前我會直接叫 ChatGPT 幫我寫完整內容，現在我反而先叫它幫我拆工作流程。
3. 用 AI 寫程式一陣子後，我覺得最重要的技能不是 prompt，而是知道什麼時候該停下來自己看。
4. 最近很多人在講 Agent，但我覺得大多數人其實只需要一個很穩的半自動流程。
5. 如果你覺得 AI 工具很多很亂，可以先只問一件事：它有沒有讓你少做一次重複判斷？

### 長 thread 大綱

1. Hook：我連續測了幾種 AI Agent workflow，最後發現最有用的是最不酷的那一種。
2. 問題：多數人一開始想做全自動，結果卡在登入、資料品質、錯誤處理與驗證。
3. 案例：用人工整理公開貼文，再讓 AI 做 Hook、情緒、CTA、爆紅原因分析。
4. 方法：先建立固定 JSON 欄位、評分機制與 Markdown 報告格式。
5. 結論：先跑順 3-7 天，再決定哪些步驟值得自動化。
6. CTA：如果你也在做 AI workflow，可以先從一個每天真的會用的小流程開始。
