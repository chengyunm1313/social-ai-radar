# 使用說明：Social AI Radar

這份文件說明如何用 Codex Desktop 與本專案腳本，從公開 X 貼文整理 AI / Tech 爆文分析報告。

目前主要支援 X 的「AI Agent」搜尋分析；Threads、Facebook 可先用人工整理資料，再請 Codex 依同一套分析框架補充報告。

## 1. 專案目標

Social AI Radar 用來找出高傳播性的 AI / Tech 內容，並拆解：

1. 熱門主題
2. 爆文候選
3. Hook 分析
4. 趨勢分析
5. 可模仿內容

執行限制：

- 只讀取公開貼文。
- 不按讚、留言、轉貼、追蹤。
- 不讀取私人訊息。
- 不修改帳號設定。
- 不儲存登入資訊。
- 不捏造互動數據或貼文內容；資料不足時要明確標示。

## 2. 開啟專案

在 Codex Desktop 開啟這個資料夾：

```text
/Users/hsuhsiang/Desktop/project/social-ai-radar
```

確認終端機工作目錄在專案根目錄後，先安裝依賴：

```bash
npm install
```

本專案需要 Node.js 20 以上。

## 3. 主要輸入與輸出

常用檔案：

- `input/x-ai-agent-raw.json`：X 原始貼文資料，分析腳本會讀取這個檔案。
- `input/manual-x.txt`：手動從 X 複製的頁面文字，可用來匯入成 raw JSON。
- `output/social-radar-report.md`：主要分析報告。
- `output/debug-x-text.txt`：Playwright 擷取時保存的頁面文字，方便排查。
- `output/debug-x-page.html`：Playwright 擷取時保存的 HTML。
- `output/debug-x-screenshot.png`：Playwright 擷取時保存的截圖。
- `knowledge/hook-library.md`：可累積的 Hook pattern。
- `prompts/social-radar.md`：可重用的分析提示詞。

## 4. 推薦流程 A：連接已登入 Chrome 擷取 X

這是目前最接近半自動的流程。它會連到你手動開啟並已登入的 Chrome，只讀取公開搜尋結果。

### 步驟一：啟動可被 Playwright 連線的 Chrome

如果 Chrome 目前已經開著，macOS 可能只會把指令交給既有瀏覽器工作階段，導致 `9222` 沒有真的啟動。建議使用下面任一種方式。

做法 A：完全關閉所有 Chrome 視窗後，執行：

```bash
/Applications/Google\ Chrome.app/Contents/MacOS/Google\ Chrome --remote-debugging-port=9222
```

做法 B：保留原本 Chrome，另外開一個獨立 debug profile：

```bash
open -na "Google Chrome" --args --remote-debugging-port=9222 --user-data-dir="$HOME/.social-ai-radar/chrome-debug"
```

做法 B 會開一個新的 Chrome 個人資料，因此需要在該視窗手動登入 X。

可以用以下指令確認 `9222` 是否已啟動：

```bash
lsof -nP -iTCP:9222 -sTCP:LISTEN
```

看到 Chrome 正在 listen 後，再執行 `npm run radar:x`。

### 步驟二：執行 X 擷取與分析

回到專案根目錄執行：

```bash
npm run radar:x
```

這個指令會依序執行：

```bash
npm run scrape:x
npm run analyze
```

`scrape:x` 會開啟 X 搜尋：

```text
https://x.com/search?q=AI%20Agent&src=typed_query&f=top
```

預設會等待 60 秒，讓你手動登入、處理驗證或調整頁面。接著腳本會滾動頁面並收集公開貼文，輸出到：

```text
input/x-ai-agent-raw.json
```

`analyze` 會讀取 `input/x-ai-agent-raw.json`，輸出：

```text
output/social-radar-report.md
```

### 可調整參數

需要延長登入等待時間或改滾動次數時，可用環境變數：

```bash
SOCIAL_RADAR_WAIT_MS=90000 SOCIAL_RADAR_SCROLL_TIMES=8 npm run radar:x
```

常用參數：

- `SOCIAL_RADAR_WAIT_MS`：等待手動登入或頁面調整的毫秒數，預設 `60000`。
- `SOCIAL_RADAR_SCROLL_TIMES`：滾動次數，預設 `5`。
- `SOCIAL_RADAR_SCROLL_DELAY_MS`：每次滾動後等待毫秒數，預設 `2000`。
- `SOCIAL_RADAR_CDP_URL`：Chrome CDP 位置，預設 `http://127.0.0.1:9222`。

## 5. 推薦流程 B：手動複製 X 頁面文字再分析

如果 Playwright 擷取不到結構化貼文，或 X 頁面限制較多，建議使用手動匯入流程。

### 步驟一：複製公開搜尋結果文字

1. 用你平常的 Chrome 登入 X。
2. 搜尋 `AI Agent`，切到熱門或最新結果。
3. 只停留在公開可見內容。
4. 選取頁面中候選貼文文字並複製。
5. 貼到：

```text
input/manual-x.txt
```

### 步驟二：匯入並分析

執行：

```bash
npm run radar:hybrid:x
```

這個指令會依序執行：

```bash
npm run import:manual:x
npm run analyze
```

`import:manual:x` 會把 `input/manual-x.txt` 轉成：

```text
input/x-ai-agent-raw.json
```

接著 `analyze` 產出：

```text
output/social-radar-report.md
```

## 6. 只分析既有資料

如果已經有 `input/x-ai-agent-raw.json`，只想重新產報告：

```bash
npm run analyze
```

適合在你手動修正 raw JSON、補上 metrics 或 URL 之後使用。

## 7. 用 Codex Desktop 下任務

你可以直接請 Codex 執行：

```text
請依照本專案規則，只讀取公開 X 貼文，執行 npm run radar:x，產出 output/social-radar-report.md。不要按讚、留言、轉貼、追蹤，也不要讀私人訊息。
```

若要走手動匯入流程：

```text
我已把 X 搜尋頁文字貼到 input/manual-x.txt。請執行 npm run radar:hybrid:x，並檢查 output/social-radar-report.md 是否包含熱門主題、TOP 爆文、爆文分析、Hook Pattern、情緒分析、可模仿內容、建議發文方向。
```

若要補強 Hook library：

```text
請根據 output/social-radar-report.md，把可重複使用的 Hook pattern 補到 knowledge/hook-library.md。不要複製原貼文，只整理結構。
```

若要產生內容草稿：

```text
請根據 output/social-radar-report.md 產出 5 篇適合 Threads 的繁體中文 AI 趨勢貼文草稿。保留 Hook 結構，但不要抄原文。
```

## 8. 報告品質檢查

每次產出 `output/social-radar-report.md` 後，建議檢查：

- 是否明確標示資料來源、擷取時間與原始資料檔。
- 是否只使用公開貼文。
- 是否包含熱門主題、爆文候選、Hook 分析、趨勢分析、可模仿內容。
- 是否有 TOP 爆文排序與 signal score。
- Hook、情緒、CTA 或爆紅原因是否可被複用。
- 可模仿內容是否沒有逐字複製原文。
- 若資料不足，是否清楚說明不足原因，而不是硬湊結論。

## 9. 常見問題

### 無法連到 Chrome remote debugging

若 `npm run radar:x` 顯示無法連到 `http://127.0.0.1:9222`，通常代表 Chrome 沒有真的帶著 `--remote-debugging-port=9222` 啟動。

先確認是否有程序監聽：

```bash
lsof -nP -iTCP:9222 -sTCP:LISTEN
```

如果沒有輸出，可完全關閉 Chrome 後重新啟動：

```bash
/Applications/Google\ Chrome.app/Contents/MacOS/Google\ Chrome --remote-debugging-port=9222
```

或另外開一個獨立 debug profile：

```bash
open -na "Google Chrome" --args --remote-debugging-port=9222 --user-data-dir="$HOME/.social-ai-radar/chrome-debug"
```

也可以改用手動匯入流程：

```bash
npm run radar:hybrid:x
```

### 報告顯示資料不足

可能原因：

- X 尚未登入。
- 搜尋頁沒有載入公開貼文。
- 等待時間太短。
- X 頁面 DOM 改版，腳本抓不到 article。

可先查看：

```text
output/debug-x-text.txt
output/debug-x-screenshot.png
```

如果 debug 檔有可讀貼文，但 raw JSON 沒有貼文，改走手動匯入流程通常最快。

### 想分析不同關鍵字

目前腳本預設關鍵字是 `AI Agent`。若要分析其他主題，可以先用 X 搜尋頁手動切換關鍵字，再用手動匯入流程；或修改 `scripts/scrape-x.js` 裡的 `SEARCH_URL` 與 `scripts/import-manual-x.js` 裡的 `KEYWORD`。

## 10. 每日固定操作

建議每日流程：

```bash
npm run radar:x
```

若擷取失敗：

```bash
npm run radar:hybrid:x
```

完成後閱讀：

```text
output/social-radar-report.md
```

再視需要請 Codex 產出 Threads 草稿、X thread 大綱或補充 `knowledge/hook-library.md`。
