# 使用說明：用 Codex Desktop 操作 Social AI Radar

這份文件說明如何用 Codex Desktop 協助執行 Social AI Radar，從公開社群貼文整理 AI / Tech 爆文分析報告。

## 1. 開啟專案

1. 打開 Codex Desktop。
2. 選擇或開啟這個專案資料夾：

```text
/Users/hsuhsiang/Desktop/project/social-ai-radar
```

3. 確認 Codex 的工作目錄在本專案根目錄。

## 2. 基本使用方式

你可以直接用自然語言請 Codex 執行任務，例如：

```text
幫我分析今天 X、Threads、Facebook 上 AI / Tech 相關的公開熱門貼文，產出 Markdown report 和 JSON raw data。
```

或指定更明確的任務：

```text
請根據 raw-posts.json 產出一份 social-radar-report.md，包含熱門主題、爆文候選、Hook 分析、趨勢分析、可模仿內容。
```

## 3. 建議工作流程

### 步驟一：收集公開貼文

請 Codex 協助整理公開貼文資料，並存成 JSON：

```text
請只讀取公開貼文，整理 AI / Tech 相關高互動內容，輸出到 raw-posts.json。
```

資料建議包含：

- 平台：X、Threads、Facebook
- 貼文網址
- 作者公開名稱
- 發文時間
- 互動數據
- 貼文內容摘要
- 可能的 Hook pattern

### 步驟二：產出分析報告

請 Codex 根據原始資料產出 Markdown 報告：

```text
請根據 raw-posts.json 產出社群爆文分析報告，檔名為 social-radar-report.md。
```

每次報告都應包含：

1. 熱門主題
2. 爆文候選
3. Hook 分析
4. 趨勢分析
5. 可模仿內容

### 步驟三：補強內容模板

如果要把爆文結構轉成可用的內容靈感，可以請 Codex：

```text
請把本次爆文候選整理成 10 個可模仿的貼文模板，保留 Hook 結構，但不要抄原文。
```

## 4. 常用指令範例

### 產出每日報告

```text
請分析今天公開的 AI / Tech 熱門貼文，更新 raw-posts.json，並產出 social-radar-report.md。
```

### 只分析既有資料

```text
請不要上網，直接根據 raw-posts.json 產出最新分析報告。
```

### 擴充 Hook library

```text
請根據這次報告，把可重複使用的 Hook pattern 補到 knowledge/hook-library.md。
```

### 產生內容草稿

```text
請根據 social-radar-report.md 產出 5 篇適合 Threads 的繁體中文 AI 趨勢貼文草稿。
```

## 5. 安全與帳號限制

使用 Codex Desktop 操作本專案時，請遵守以下原則：

- 只讀取公開貼文。
- 不操作按讚、留言、轉貼、追蹤。
- 不讀取私人訊息。
- 不修改任何社群帳號設定。
- 不儲存登入資訊。
- 不抄襲原文，只分析結構與可模仿框架。

## 6. 建議檔案用途

- `README.md`：專案目標與限制。
- `AGENTS.md`：Codex 執行本專案時應遵守的代理規則。
- `raw-posts.json`：原始貼文資料。
- `social-radar-report.md`：主要分析報告。
- `knowledge/hook-library.md`：可重複使用的 Hook pattern。
- `prompts/social-radar.md`：可重用的分析提示詞。
- `output/sample-report.md`：報告範例。

## 7. 品質檢查

每次請 Codex 產出報告後，建議檢查：

- 是否明確標示資料來源平台。
- 是否只使用公開貼文。
- 是否包含五個必要章節。
- Hook 分析是否可被複用。
- 可模仿內容是否沒有直接複製原文。
- Markdown 格式是否清楚可讀。
